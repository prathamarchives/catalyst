Brain status

Catalyst teaches agents what you'd approve, reject, revise, and remember. The loop below is the product — this panel just makes it visible.

task→load brain→produce→review vs standards→show you→capture feedback→update brain

Guided onboarding

Local-first: this builds markdown files under outputs/<name>/ on your machine from your own typed answers. No scanning happens on this screen, and nothing leaves your computer.

Excluded by default before any scan: secrets, tokens, private DMs, client data, binaries, vendor/build folders, dependency folders. Discovery (Sources tab) is read-only path metadata; reading file contents needs your approval.

Source approval

Read-only discovery finds categories of source material — never giant path dumps, never contents. You approve a scope before anything is read.

One compact approval question (cautious mode):
"Approve recommended safe scan, edit scope, or manual mode?"
If your starting prompt already authorized a safe scan, an agent proceeds in autonomous authorized mode without re-asking. This panel never reads file contents on its own.

Brain explorer

Real markdown under outputs/<name>/catalyst-brain/. Each file shows its purpose, when agents load it, and what it affects. Edits save straight to disk.

Select a brain, then a file.

Proof / evaluation

The differentiator: an output is unverified until it's checked against your written standard. This shows which brain files load and what feedback would update.

uses BYOK if a key is set, otherwise a deterministic local checklist (mock)

BYOK — bring your own key

Optional. Everything above works with no key (mock mode). A key only powers AI-assisted synthesis and review.

What gets sent if you enable BYOK: only the text you pass to an assisted action (onboarding answers, a draft you submit for review) is sent to your chosen provider over the network. Nothing is sent in mock mode. The key is read from an environment variable only — never typed into this page, never stored by the panel, never committed.
