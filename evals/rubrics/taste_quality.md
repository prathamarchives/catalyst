# rubric — taste / protocol quality (manual)

Score the repo as an artifact: does it carry its own thesis, or does it read like the thing it's against? Score each 1–5 (1 = embarrassing, 3 = acceptable, 5 = exemplary). Best scored by someone who didn't write the files.

| # | question | score (1–5) | notes |
|---|---|---|---|
| 1 | Does the repo feel open-source useful, or startup vapor? | | |
| 2 | Does it avoid corporate AI-wrapper voice in its own prose? | | |
| 3 | Do rejected examples feel central to the system (not an afterthought file)? | | |
| 4 | Is "same model, different brain" made obvious — especially by the before/after example? | | |
| 5 | Does it preserve raw user language in the example brain (quotes survive, unsanitized)? | | |
| 6 | Does it avoid reducing the product to "write like me" (judgment/context/memory visible, not just voice)? | | |
| 7 | Would a creator/operator who uses agents daily find this genuinely useful tomorrow? | | |

## scoring

- 30–35: the repo carries the thesis
- 22–29: solid; sharpen the lowest items before wide sharing
- below 22: it's contradicting itself somewhere — usually voice (check the README and templates against anti-slop standards)

## the one-sentence test

After reading only README.md, can a stranger say what this does and why it matters in one sentence — and is that sentence about *brains/context*, not "AI writing tool"? If not, the README failed regardless of other scores.
