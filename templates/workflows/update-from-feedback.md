# workflow — update from feedback

Run every time the user reacts to creative output. This is the compounding step — see [prompts/07-update-from-feedback.md](../../prompts/07-update-from-feedback.md) for the full protocol.

## steps

1. **Capture the raw correction verbatim.** Their phrasing is the data.
2. **Extract the rule.** What preference does this reveal beyond this one output?
3. **Append to feedback-memory.md** — dated, raw quote + distilled rule + files patched.
4. **Patch the affected brain files** (voice / taste / judgment / anti-slop / lexicon). A rule that lives only in feedback-memory gets missed.
5. **Check for recurrence.** Same correction twice = promote to standing laws in feedback-memory.md and to anti-slop.md or judgment.md.
6. **Update the skill** if operating behavior changed (new red flag, new pre-final check).
7. **If the user rewrote your output:** diff it. Extract at least one rule from their changes.

## rules

- record feedback even when it stings; never argue with it in the files
- contradictions are signal: note both rules, ask which wins, record the ruling
- date everything; newer beats older
