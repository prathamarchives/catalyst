# creative-brain templates

These ten files are the blueprint for a Creative Brain. Agents: copy the structure into `outputs/<name>/creative-brain/` and fill it with the user's extracted material. **Never edit these masters.**

## The ten files

| file | role |
|---|---|
| identity.md | who the user is, what they're building, what they are not |
| context.md | live state: projects, goals, audience, constraints |
| voice.md | how they actually talk/write, with evidence |
| taste.md | what they like and notice — by contrast |
| judgment.md | their decision rules: good / weak / too safe / too much |
| anti-slop.md | banned words, structures, vibes |
| references.md | what to learn from each reference, what not to copy |
| rejected-examples.md | failed outputs and why — the gold file |
| feedback-memory.md | every correction, distilled into durable rules |
| lexicon.md | their words, phrases, shorthand, mission lines |

## Why ten files instead of one

Different creative moments need different files. A reply needs voice + lexicon; a design brief needs taste + references; a final review needs judgment + anti-slop + rejected-examples. Splitting them keeps each load small and each update targeted.

## The two files that matter most

- **rejected-examples.md** — negative taste is half the brain. An agent that knows what the user kills stops producing it.
- **feedback-memory.md** — this is what makes the system compound. Corrections become rules; rules persist across sessions and agents.

Every template below specifies: purpose, what belongs, what doesn't, suggested sections, example entries, and its update rule.
