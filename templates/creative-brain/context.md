# context.md — template

## purpose

The live state an agent needs before doing creative work: what's happening now, for whom, under what constraints. Identity is who they are; context is what's true this month.

## what belongs here

- current projects and their stage
- current goals (and the one that matters most right now)
- audience: who the work is for, where they live, what they respond to
- offers/products if any
- constraints: time, money, tools, platform rules
- live priorities and anything recently deprecated ("we stopped doing X")

## what does not belong here

- permanent identity traits (identity.md)
- style rules (voice/taste/anti-slop)
- old projects kept for nostalgia — context is current or it's noise

## suggested sections

```md
## now (updated: <date>)
## goals
## audience
## constraints
## recently dead
```

## example entries

```md
## now (updated: 2026-06-12)
launching the spring collection; newsletter is the main channel this quarter.

## recently dead
- stopped weekly youtube uploads — don't reference "this week's video"
```

## update rule

Fast file. Update at the start of any significant work session or when the user says priorities changed. Always refresh the `updated:` date — stale context flagged as current is worse than no context.
