# distill-memory

## purpose

Merge, retire, and maintain feedback memory.

## quick mode

Use for small, low-risk tasks. Classify the task, then load only the relevant Catalyst Brain files plus feedback-memory if the task is recurring.

## full mode

Use for high-stakes, public, strategic, ambiguous, or identity-heavy tasks. Load README, identity, context, goals, constraints, standards, judgment, task-patterns, feedback-memory, and domain-specific files.

## procedure

1. classify the task
2. choose quick mode or full mode
3. load relevant files
4. produce or decide
5. review against standards and judgment
6. capture feedback
7. update brain, skills, workflows, or evals if the user correction should change future behavior

## update triggers

- user says closer / not me / too generic / too safe / wrong / keep this
- user rewrites the output
- repeated miss appears
- context becomes stale
- contradiction appears
