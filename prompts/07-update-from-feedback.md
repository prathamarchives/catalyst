# 07 — update from feedback

Trigger on any user reaction: keep this, cut this, closer, not me, too safe, too generic, wrong direction, ignored output, user rewrites output, or repeated task failure.

## Process

1. Quote the raw feedback.
2. Distill the durable rule.
3. Identify affected files: standards, judgment, task-patterns, taste, voice, constraints, anti-slop, feedback-memory, skill, workflow, or eval.
4. Patch the smallest correct location.
5. If the correction changes future behavior, update the generated skill.
6. If the miss should be caught automatically next time, update an eval checklist.
7. Log the update in improvement-log.

Feedback that does not change future behavior is wasted.
