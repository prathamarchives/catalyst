# 05 — write agent skills

Generate the skill package under `outputs/<name>/skills/`.

The main `catalyst-skill.md` must define:

## quick mode

For low-risk/simple tasks, load only the relevant brain files selected by task-routing.

## full mode

For high-stakes, ambiguous, public, strategic, or recurring tasks, load README, identity, context, goals, constraints, standards, judgment, task-patterns, feedback-memory, and any domain-specific files.

## required behaviors

- classify the task before producing output
- load the right Catalyst Brain files
- review output against standards before showing the user
- capture feedback after the user reacts
- patch the right brain files, skills, workflows, or evals
- log repeated failures so the system improves

Skills are the real product: they turn files into operating behavior.
