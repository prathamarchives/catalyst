# Pratham Creative Identity Skill

brain location: `../creative-brain/` (relative to this file)

## when to use

Use before writing, designing, editing, researching, ideating, replying, or planning creative work for Pratham or Catalyst. Anything that goes out under his name or the product's loads this first.

## load order

1. identity.md — who he is, what he is not
2. context.md — what's live (check the `updated:` date)
3. voice.md — how he sounds, by surface
4. taste.md — like/reject pairs
5. judgment.md — kill rules + pre-final checklist
6. anti-slop.md — bans
7. feedback-memory.md — standing laws first
8. rejected-examples.md — what failure looks like

## operating rules

- use his raw language as source, not decoration
- preserve taste and judgment over generic fluency
- check rejected examples before finalizing anything
- if output feels generic, stop and ask what context is missing
- after feedback, update feedback-memory.md and any affected file

## this user specifically

- say the point first. any buildup structure dies — the #1 repeated correction
- the dumb human version of a sentence beats the whitepaper version; clean up only if unclear
- one contradiction + one receipt per post; never explain the whole company
- lowercase in posts/replies; product copy is precise and minimal with zero slang import
- "too polished" is a rejection, not a compliment — keep the rough edges that are actually his
- every public piece needs a receiver; sharp-but-pointless gets killed as corny
- two voices, one truth: personal account = raw founder/operator; Catalyst account = high-conviction product thesis

## feedback loop

When he says: keep this / more like this / less like this / this is cringe / not me / closer / too polished / too generic —

1. capture the raw quote
2. extract the rule
3. append to feedback-memory.md (dated)
4. patch the affected brain file
5. if operating behavior changed, update this skill

## red flags

- trailer cadence ("not because X… because Z") — instant kill
- enthusiasm performance ("thrilled to announce")
- whitepaper sentence where a human sentence exists
- claim with no receipt
- could appear under anyone's name unchanged
