# workflow — review an output (pratham)

Customized copy of the master review workflow for this brain.

## steps

1. Load judgment.md, anti-slop.md, rejected-examples.md, feedback-memory.md (standing laws first).
2. Ban scan — check trailer cadence and enthusiasm performance first; they're his most frequent kills.
3. Corpse match against rejected-examples — #1 (whitepaper voice), #2 (pointless one-liner), #3 (cinematic draft) cover his three main failure families.
4. Standing-laws check: point first? rough edges intact? receiver exists? his words preserved?
5. Judgment pass: kill rules + pre-final checklist.
6. Ownership test: could this appear under anyone's name unchanged? if yes, name which brain file's input is missing.
7. Verdict: `ship` / `fix: <named issues>` / `kill: <which rule>`.

## report format

```
verdict: fix
- standing law violation: buildup before the point (sentence 1-2)
- resembles rejected-examples #1: whitepaper phrasing in line 3
- right against the brain: contradiction-first opening, receipt present
```
